## 课题背景

本题面向公司内 AI Coding 场景下的大模型在线推理系统优化问题，目标是在 8 卡 H100 集群上完成 Deepseek-v4-flash 的稳定部署与性能优化，并在真实 SWE-bench 工作负载下进行系统级验证。

参赛者需要面对的核心问题包括：

1. **TP/DP 切分策略协调** — 在长上下文（代码仓库级依赖）与高并发请求并存的情况下，如何协调 TP/DP 切分策略
2. **KV Cache 生命周期管理** — 长上下文堆叠导致显存压力
3. **Prefill/Decode 阶段调度冲突** — GPU 利用率与延迟之间的矛盾
4. **长尾分布与突发流量** — 请求分布具有明显的长尾特征与突发性，容易引发：
   - 延迟抖动
   - 显存压力上升（OOM 风险）
   - 吞吐下降

该任务不局限于单点推理优化，而是强调从模型部署、请求链路、资源调度到执行层优化的**端到端工程能力**。

## 交付物建议

以可复现的工程化方案为核心，包括：

- [x] **可直接运行的容器镜像**（Docker Image，用于一键拉起 8×H100 推理服务）
- [x] **完整部署与启动脚本**（包含模型加载、并行策略配置与服务启动参数）
- [x] **关键推理优化代码**（如 KV Cache 管理、batching / scheduling / speculative 或 chunked prefill 等策略实现）
- [x] **必要的配置说明与实验复现文档**（用于支持在 SWE-bench 数据集上的稳定评测与对比分析）

整体要求以工程可落地性为主，保证评测环境中可**一键部署**、**可重复运行**、**可量化评估**。

- 部署环境：Docker 容器

## 模型约束

- 量化精度：FP8（预期）

## 评测基准

- 数据集：SWE-bench（Software Engineering Benchmark）
- 工作负载特性：
  - 长尾分布
  - 突发性
  - 长上下文依赖（代码仓库级）

---

## 核心优化方向

| # | 方向 | 目标 |
|---|------|------|
| 1 | KV Cache 管理 | 防止显存溢出，提升缓存命中率 |
| 2 | Prefill/Decode 调度 | 消除长尾请求阻塞，抗突发 |
| 3 | TP=8 张量并行 | 分布式推理，跨卡通信优化 |
| 4 | 动态批处理 | 吞吐最大化，延迟最小化 |
| 5 | FP8 量化 | 显存与精度平衡 |
