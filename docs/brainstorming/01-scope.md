## 决策结论

### 参赛版 Scope

| 模块 | 功能 | 决策 | 理由 |
|------|------|------|------|
| 控制层（自动调优） | 实时自动调优循环 | 🔴 **砍掉** | 4小时赛程不可能收敛，预留接口/mock |
| 推理引擎层 | FP8 + vLLM | ✅ **必须有** | 核心 |
| 路由调度层 | Chunked Prefill | ✅ **必须有** | 消除长尾抖动的核心技术 |
| 路由调度层 | 动态批处理 | ✅ **必须有** | 吞吐最大化的关键 |
| 路由调度层 | KV Cache 动态淘汰 | ✅ **必须有** | 防 OOM 的关键 |
| 可观测层 | Prometheus 指标 | ✅ **必须有** | 评测必需 |
| 可观测层 | Nsight Systems | ⚠️ **降级** | 改为文档说明，不装入容器 |
| 实验文档 | 复现文档 | ✅ **必须有** | 评分依据 |

### 核心策略

**"工程可落地优先，参赛亮点突出"**

砍掉控制层自动调优，聚焦两个核心能力：
1. **调度优化** — Chunked Prefill + 动态批处理
2. **显存管理** — KV Cache 动态淘汰 + FP8 量化

## 架构分层策略

### 参赛版简化架构

```
控制层（Auto-tuner）  →  API Gateway（预留接口，运行时手动调参）
         ↓
推理引擎层（vLLM）   →  真实执行
         ↓
路由调度层            →  Chunked Prefill + 动态 Batching
         ↓
可观测层              →  Prometheus Metrics
```

### Mock 策略

- 本地无卡环境：Mock vLLM，返回模拟延迟
- 参赛环境：切换真实 vLLM，其他模块不动
