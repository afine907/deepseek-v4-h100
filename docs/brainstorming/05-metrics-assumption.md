## 性能指标假设

### 延迟指标

| 指标 | 基线（假设） | 优化目标 | 提升幅度 | 备注 |
|------|------------|---------|---------|------|
| 平均延迟 (Mean) | 3.0 s | < 2.0 s | 33%+ |  |
| P50 延迟 | 1.5 s | < 1.0 s | 33%+ |  |
| P90 延迟 | 5.0 s | < 3.0 s | 40%+ |  |
| P99 延迟 | 10.0 s | < 5.0 s | 50%+ | 长尾优化重点 |
| 最大延迟 (Max) | 30.0 s | < 15.0 s | 50%+ | 防 OOM 关键指标 |

**假设依据：**
- DeepSeek-V4-Flash 单次生成 token 数预估：平均 200~500 tokens
- H100 FP8 单 token decode 约 5~10ms
- 长尾请求（>16k tokens）P99 延迟预计 8~12s

### 吞吐指标

| 指标 | 基线（假设） | 优化目标 | 提升幅度 | 备注 |
|------|------------|---------|---------|------|
| QPS（查询/秒） | 50 | > 100 | 100%+ | 翻倍 |
| Tokens/sec | 10,000 | > 20,000 | 100%+ |  |
| GPU 利用率 | 50% | > 80% | 30pp | 参赛亮点 |

**假设依据：**
- 8×H100 理论算力：约 16,000 TFLOPS (FP8)
- 基线预计利用率 50%（长尾请求导致 GPU 空闲）
- 优化后目标 80%+

### 显存指标

| 指标 | 基线（假设） | 优化目标 | 备注 |
