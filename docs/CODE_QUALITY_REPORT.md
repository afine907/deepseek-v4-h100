## 二、TDD 践行评估

### 结论：未践行 TDD

源代码与测试**分开提交**，间隔仅 6 分钟，是典型的 "code-first, test-second" 而非 TDD：

| 提交 | 时间 | 内容 |
|------|------|------|
| `9d86cfd` | 11:29 | `feat(core): implement hexagonal architecture ports + adapters`（无测试） |
| `f18917b` | 11:35 | `test: add unit and integration tests for core components — Task 4`（6 分钟后） |

### 反 TDD 模式：测试记录 Bug 而非指定行为

**`tests/unit/test_scheduler.py:474-502`**
```python
def test_two_chunk_request_needs_two_prefill_slots_across_steps(self):
    """
    Currently BUGGY: current_chunk never increments
    ...
    BUG EXPECTED TO FAIL HERE
    """
    # 无 assert — 测试只记录 bug，不验证修复
```

**`tests/unit/test_scheduler.py:509-530`**
```python
def test_duplicate_request_id_overwrites_running(self):
    """
    This is a known limitation — no error is raised
    """
    # 测试文档化已知限制，而不是指定期望行为
```

TDD 要求测试**先于代码**，并作为行为的规范说明（specification）。当前测试更接近**事后验证**。

### 3.2 关键 Bug

#### Bug 1：`current_chunk` 从不递增（多 Chunk 请求卡死）

**位置**：`src/core/scheduler.py:135-148`（`_submit_batch` 方法）

多 Chunk 请求提交后，`pr.current_chunk` 永远停在 0，导致每次 `step()` 都重复提交同一个 Chunk。

**测试记录**：`tests/unit/test_scheduler.py:474` 测试本身标注 "BUGGY"，但没有修复。

#### Bug 2：`set_max_blocks` 无锁写入竞态

**位置**：`src/core/kv_cache_manager.py:38-40`

```python
def set_max_blocks(self, max_blocks: int) -> None:
    self._max_blocks = max_blocks  # 未持有 self._lock
```

所有读取路径（`get_memory_usage`、`_evict_locked` 等）都持有锁，唯独写入路径没有。并发调用时存在 data race。

#### Bug 3：空 Prompt 返回 `['']` 而非空数组

**位置**：`src/core/scheduler.py:81`

```python
return chunks if chunks else [prompt]  # 空 prompt → [""]
```

空 Prompt 被当作 1 个 decode-only Chunk 处理，违反最小惊讶原则。`test_empty_prompt_returns_one_chunk` 将此行为 codified，无法体现这是设计缺陷。

### 4.2 并发测试过弱

`tests/unit/test_kv_cache_manager.py:291-357`（`TestConcurrency`）只断言：

```python

assert no_exception
assert total_count == n_threads * iterations
```

这无法验证互斥正确性——即使锁是 no-op，只要不抛异常就能通过。

### 4.3 Flaky 测试

`src/adapters/mock_adapter.py:50-51` 使用无种子的随机数：

```python
latency = random.gauss(mean_latency_ms, std_latency_ms)
chunk_count = random.randint(1, 4)
```

时序敏感的测试会间歇性失败（当前有 3 个因此失败）。
